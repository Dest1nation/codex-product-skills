# Questioning Strategy Policy

Ask questions only when they unlock product design.

In Discovery Conversation, questions should help the user think. They are not a form-filling sequence.

## Good Questions

- Change scope.
- Clarify the main role or scenario.
- Decide whether a state machine is needed.
- Determine a permission boundary.
- Identify a system dependency.
- Affect page structure or acceptance.

## Avoid

- Asking for every field before scenario is known.
- Asking implementation details before product rules exist.
- Asking multiple unrelated questions at once.
- Re-asking information already provided in different words.
