# Full End-to-End Example

1. Discovery Conversation helps the user move from "feedback is scattered and hard to follow up" to a clearer understanding: the first priority is making feedback handling traceable.
2. Decision Readiness reaches High because the requirement goal, core user, core scenario, and main boundary are clear enough.
3. Discovery Summary creates `brief_material`: goal, core user, core scenario, main boundary, and unresolved questions.
4. Structured IR consumes the accepted boundary and designs business flow, system flow, states, permissions, rules, and prototype implications.
5. Scenario Gap Handler classifies missing cases: in-boundary gaps are added to the structure, out-of-scope scenarios become future phase notes, and direction-changing gaps return to Discovery.
6. Handoff IR packages the structured design into PRD input materials and keeps assumptions, open questions, and scenario gap handling visible for `write-prd`.
